# 架构图

## 0. 当前阶段

- 阶段 0 已于 2026-07-10 验收通过。
- 阶段 1 候选版已实现本机匿名身份、IndexedDB、文字记录、时间线、本地回应引擎和离线 PWA；前两版视觉未通过，当前编辑式去卡片化返工候选版等待用户再次验收。
- 官方核心长期只实现离线与 BYOK 两条路径；服务器、账户、权限、云存储和同步只维护开放契约，不成为默认依赖。
- 本文描述的是完整演进架构，不代表后续 AI 与开放扩展模块已经实现。阶段 1 的实际实施与验收边界以 [`04-阶段1设计.md`](04-阶段1设计.md) 为准。

## 1. 总体架构

第一版采用本地优先架构。PWA 是完整应用，AI 和云端同步都是可插拔能力。

```mermaid
flowchart TB
    U["用户"] --> UI["PWA 界面"]

    subgraph DEVICE["用户设备：默认数据边界"]
        UI --> FLOW["陪伴流程控制器"]
        UI --> RECORD["记录服务"]
        UI --> PROFILE["用户与偏好服务"]

        FLOW --> LOCALRESP["本地回应引擎"]
        FLOW --> AIGATE["AI 能力门"]

        RECORD --> DB["IndexedDB"]
        PROFILE --> DB
        LOCALRESP --> DB

        UI --> IMAGE["图片预处理\n压缩、去元数据"]
        IMAGE --> DB

        AIGATE --> PROVIDERS["AI Provider Adapter"]
        KEY["本机 Key Vault"] --> PROVIDERS

        SYNC["Sync Adapter\n默认关闭"] <--> DB
    end

    PROVIDERS -->|"用户明确选择后"| QWEN["千问视觉模型"]
    PROVIDERS -->|"可选备选"| OR["OpenRouter 免费模型"]
    PROVIDERS --> MOCK["Mock / 离线测试模型"]

    SYNC -.->|"配置兼容服务器后"| CONTRACT["开放扩展契约\nAuth / Permission / Sync / Storage"]
    CONTRACT -.->|"社区或自托管实现"| CLOUD["可选兼容服务器"]
    CLOUD --> AUTH["账户与认证"]
    CLOUD --> CDB["加密存储"]
```

核心约束：

- PWA 不连接任何外部服务，也能完成基础陪伴和记录。
- 图片只有在用户明确选择 AI 理解时才离开设备。
- 未配置服务器时不展示登录或同步入口，也不发送相关网络请求。
- 兼容服务器不可用不影响本地使用和 BYOK AI。
- AI 输出必须经过产品规则检查后才能展示。

## 2. 模块职责

### PWA 界面

- 展示四个核心状态入口。
- 收集用户当前意图。
- 明确区分“只记录”和“交给 AI 理解”。
- 不直接操作模型或数据库。

### 陪伴流程控制器

- 决定当前体验处于哪条分支。
- 优先调用本地回应引擎。
- 只有在用户主动请求时才开启 AI 能力门。
- 将模型结果交给安全规则检查。

### 本地回应引擎

- 无网络可运行。
- 使用人工编写、审核过的回应。
- 根据用户偏好调整称呼、长度和语气。
- 避免近期重复，不假装拥有专业医疗判断。

### 记录服务

- 保存文字、状态、用户意图和回应。
- 保存本地图片或图片引用。
- 提供时间线、导出和删除。
- 不计算连续打卡和完成率。

### 用户与偏好服务

- 管理本机匿名身份。
- 保存可选资料和表达偏好。
- 保存用户明确认可或不喜欢的回应方式。
- 登录后负责本机身份与云端身份的合并。

### AI Provider Adapter

- 统一模型请求和返回格式。
- 隔离不同供应商的接口差异。
- 支持能力检测：文本、图片、结构化输出。
- 模型失败时返回可识别错误，由流程控制器降级。

### Sync Adapter

- 官方核心只提供接口、关闭态实现和一致性测试，不提供必须部署的同步服务器。
- 只有用户配置兼容服务器并主动登录后，才上传允许同步的数据。
- 支持增量同步、冲突保留和删除传播。
- 不同步模型 Key。

### 开放扩展契约

- 定义兼容服务器的能力发现、账户、认证、权限、同步、云存储、导出和删除语义。
- 使用稳定编号、规范关键词、OpenAPI、JSON Schema、有效示例和一致性用例。
- 同一份规范同时面向人类开发者与 AI Agent，禁止依靠未写明的隐含约定。
- 社区实现可以选择不同技术栈，但必须通过相同的契约测试。
- 详细格式和兼容边界见 [`05-开放扩展接口规范.md`](05-开放扩展接口规范.md)。

## 3. 拍照数据流

```mermaid
sequenceDiagram
    actor User as 用户
    participant UI as PWA
    participant Local as 本地记录
    participant Image as 图片预处理
    participant AI as AI Provider
    participant Guard as 回应规则检查

    User->>UI: 拍照或选择图片
    UI->>Image: 压缩并移除元数据
    Image->>Local: 保存本地预览
    UI->>User: 你希望我怎么处理？

    alt 只记录
        User->>UI: 只替我记住
        UI->>Local: 保存，不发送网络请求
        UI-->>User: 已经记下了，不评价
    else 希望 AI 理解
        User->>UI: 选择理解方式并确认供应商
        UI->>AI: 发送处理后的图片和最少上下文
        AI-->>UI: 返回理解或估算
        UI->>Guard: 检查羞辱、诊断和补偿性建议
        Guard-->>UI: 通过、改写或拒绝展示
        UI->>Local: 保存用户允许保留的结果
        UI-->>User: 展示非审判回应
    end
```

## 4. 回应生成架构

```mermaid
flowchart LR
    STATE["用户当前状态"] --> POLICY["产品规则"]
    PREF["用户偏好"] --> POLICY
    HISTORY["最少必要历史"] --> POLICY

    POLICY --> DECIDE{"是否需要 AI？"}
    DECIDE -->|"否"| LOCAL["本地回应库"]
    DECIDE -->|"用户主动选择"| MODEL["模型适配器"]

    LOCAL --> CHECK["非审判规则检查"]
    MODEL --> CHECK

    CHECK -->|"通过"| SHOW["展示给用户"]
    CHECK -->|"不通过"| FALLBACK["安全的本地回应"]
    FALLBACK --> SHOW
```

产品规则优先于模型。模型不能把产品重新变成热量裁判或健身教练。

## 6. 部署形态

### 第一阶段

```text
GitHub 源代码
  └── 静态 PWA 托管
        └── 用户设备上的 IndexedDB
```

没有自有业务后端。部署失败不会导致已有本地记录丢失。

### 配置兼容服务器后

```text
静态 PWA
  ├── 可选 AI 服务（用户 Key）
  └── 可选兼容服务器
        ├── 认证
        ├── 权限
        ├── 同步与云存储
        └── 账户导出与删除
```

兼容服务器由社区、自托管者或第三方实现。官方核心只维护客户端适配器、规范和一致性测试。

### 加入原生安装后

```text
同一套 Web 核心
  ├── PWA
  ├── Capacitor Android
  └── Capacitor iOS
```

## 7. 必须守住的架构边界

- 无 Key 路径不能意外调用模型。
- “只记录”路径不能上传图片。
- 身高体重为空不能导致任何核心页面报错。
- 登录和同步模块不能成为 App 启动依赖。
- 未配置兼容服务器时不能出现账户请求、同步请求或隐藏遥测。
- 兼容服务器不能接收、同步或记录用户的模型 Key。
- 模型 Key 不进入同步、崩溃报告或分析日志。
- AI 供应商可以替换，陪伴规则不能由供应商决定。
- 专业课程、社区和排行榜不进入核心模块。
