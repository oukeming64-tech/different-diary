# 线上部署

这份说明写给第一次部署网站的人。你不需要购买服务器，也不需要把自己的 OpenRouter Key 交给部署平台。部署完成后，你会得到一个独立网址；访问者的日记、照片和运动记录仍然保存在各自的浏览器里。

## 最省事：一键部署到 Cloudflare

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/oukeming64-tech/different-diary)

1. 点击上面的按钮。
2. 登录 Cloudflare，并按提示连接 GitHub。
3. 填写新仓库名称和 Worker 名称；默认值能用就不用改。
4. 确认创建并等待构建完成。
5. Cloudflare 会给出一个 `*.workers.dev` 网址。打开它，就能使用自己的线上版本。

这个流程会在你的 GitHub 账户下创建一份项目副本，并配置后续自动部署。Cloudflare 官方对一键部署流程的说明见 [Deploy to Cloudflare buttons](https://developers.cloudflare.com/workers/platform/deploy-buttons/)。

## 已经 Fork 过：连接自己的仓库

如果你已经在 GitHub 点过 **Fork**，可以让 Cloudflare 直接监听这份仓库：

1. 登录 Cloudflare 控制台，进入 **Workers & Pages**。
2. 创建一个 Worker，并选择从 Git 仓库导入。
3. 连接 GitHub，选择你 Fork 后的 `different-diary`。
4. 生产分支选择 `main`，项目根目录保持默认。
5. 构建命令填写：

   ```bash
   npm run build
   ```

6. 部署命令填写：

   ```bash
   npx wrangler deploy --config dist/server/wrangler.json
   ```

7. 保存并开始部署。

以后向 `main` 推送更新，Cloudflare Workers Builds 就会重新构建并发布。官方的构建设置说明见 [Workers Builds configuration](https://developers.cloudflare.com/workers/ci-cd/builds/configuration/)。

## 熟悉命令行：从电脑部署

先准备 Node.js 22.15 或更高版本，然后在项目目录执行：

```bash
npm install
npx wrangler login
npm run deploy
```

`wrangler login` 会打开 Cloudflare 登录授权页；`npm run deploy` 会先构建，再把生成结果发布到当前 Cloudflare 账户。

如果只想确认部署包能否生成，不真正发布，可以运行：

```bash
npm run deploy:check
```

## 这个项目不需要填写什么

- 不需要数据库、D1、R2 或对象存储。
- 不需要服务端账户系统。
- 不要把 OpenRouter Key 写进 Cloudflare 环境变量或 GitHub Secrets；Key 由每位使用者在自己的浏览器页面里输入。
- 不需要修改 `.openai/hosting.json`。它只记录原项目的站点配置，不参与上面的 Cloudflare Workers 部署命令。

## 部署后检查

1. 打开 Cloudflare 提供的网址，确认首页出现四个状态入口。
2. 打开 `你的网址/manifest.webmanifest`，确认可以看到 PWA 清单。
3. 按 README 的“如何把它装成 PWA”操作一次。
4. 新建一条测试记录，刷新页面，确认记录仍在。
5. 不连接 OpenRouter Key 时，确认基础回应、记录、照片和运动流程仍可使用。

每个网址下的数据彼此独立。换域名、换浏览器或换设备后，原来的本机记录不会自动跟过去。

## 常见问题

### 构建时报 `registerHooks` 不存在

Node.js 版本太旧。把构建环境升级到 22.15 或更高版本，再重新部署。

### Worker 名称已经被使用

换一个只包含小写字母、数字和短横线的名字，例如 `my-different-diary`。

### GitHub Pages 能不能直接部署

当前项目会生成 Cloudflare Worker 和静态资源，不是单纯的静态网页目录，因此不建议直接使用 GitHub Pages。Cloudflare Workers 是当前经过验证的部署路径。

### 部署后能不能看到原站点的数据

不能。记录默认保存在浏览器的当前域名下；新网址会拥有一套新的本机空间。需要保留旧记录时，请先在旧站点导出 JSON 副本。
