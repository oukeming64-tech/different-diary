# 参与贡献

谢谢你愿意一起完善《不一样的日记》。这个项目欢迎代码、测试、文档、无障碍与产品建议，但所有改动都需要守住一个前提：记录可以帮助人记住生活，不能拿来评判人。

## 开始之前

- 先搜索已有 Issue，确认问题是否已经有人处理。
- 功能范围较大的提议请先开 Issue，说明它解决什么问题，以及为什么不会把记录变成新的考核。
- 不要提交真实 OpenRouter Key、个人照片、导出的日记数据或其他隐私内容。

## 本地运行

需要 Node.js 22.13 或更高版本。

```bash
npm install
npm run dev
```

提交前请运行：

```bash
npm run lint
npm test
```

## 产品边界

- 不增加羞辱、失败、超标、连续打卡、排行榜或补偿性饮食与运动文案。
- 不把登录、身体资料或模型 Key 变成首次使用的门槛。
- 照片“只记录”路径不得调用模型或上传图片。
- 新的外发行为必须在动作发生前明确告知，并由用户主动确认。
- 对数据库或导出格式的修改必须提供迁移与回归测试，不能让已有本机记录消失。

## Pull Request

请让每个 PR 只处理一个清晰主题，并在说明中写明：

- 改了什么，以及为什么。
- 对用户体验、隐私或本机数据有没有影响。
- 运行了哪些验证。
- 哪些内容明确不在本次范围内。

安全问题请按 [`SECURITY.md`](SECURITY.md) 私下报告，不要创建公开 Issue。
