# 减肥拍拍乐

一个面向“同时上班、控制体重、偶尔健身”的普通人的非审判型陪伴工具。

## 当前进度

阶段 0 手机端交互原型已于 2026-07-10 验收通过，已完成四个核心入口及各自的可点击回应闭环：

- 我想吃点东西
- 我今天不想练
- 我刚练完，很累
- 没什么，只是来坐坐

阶段 1“本地优先 MVP”已完成基础功能，但前两版视觉没有通过主观验收：第一版动效不足，第二版仍有统一大圆角卡片的 AI 模板感。当前已按 Square、Editorial、Spacious 设计规则完成去卡片化返工，等待再次体验确认。设计与验收记录见 [`docs/04-阶段1设计.md`](docs/04-阶段1设计.md)。

当前候选版已实现本机匿名身份、80 条人工回应、可选文字记录、时间线、单条删除、JSON 导出、彻底清空、PWA 安装和离线应用壳。视觉层使用极光光场、缓慢轨道和页面节奏，同时将四入口、选择、回应、时间线与数据管理改为编号目录、引文和细线列表，不再依赖重复圆角卡片。数据刷新后仍保留在当前设备。数据库以外的后端、登录、真实 AI、图片识别和云同步仍不在范围内。

## 开源实现原则

官方核心只实现两条不依赖自有服务器的路径：完全离线使用，以及用户自带 Key 使用。账户、认证、权限、云存储和同步作为可选开放扩展，不成为应用启动或核心体验的依赖。

项目会维护版本化的开放扩展契约、OpenAPI/JSON Schema 和一致性测试，方便社区开发者或 AI Agent 实现兼容服务；官方核心不预置一套必须部署的业务后端。规范总则见 [`docs/05-开放扩展接口规范.md`](docs/05-开放扩展接口规范.md)。
