# 减肥拍拍乐

一个面向“同时上班、控制体重、偶尔健身”的普通人的非审判型陪伴工具。

## 当前进度

阶段 0 手机端交互原型已于 2026-07-10 验收通过，已完成四个核心入口及各自的可点击回应闭环：

- 我想吃点东西
- 我今天不想练
- 我刚练完，很累
- 没什么，只是来坐坐

阶段 1“本地优先 MVP”首个候选版已完成，但用户认为视觉不够酷炫、放松，动画效果不足，因此没有通过主观验收。当前已进入动态视觉返工候选版，等待再次体验确认。设计与验收记录见 [`docs/04-阶段1设计.md`](docs/04-阶段1设计.md)。

当前候选版已实现本机匿名身份、80 条人工回应、可选文字记录、时间线、单条删除、JSON 导出、彻底清空、PWA 安装和离线应用壳，并加入极光光场、缓慢轨道、页面转场、错峰卡片、回应柔光、保存涟漪和时间线浮现等本地 CSS 动效。数据刷新后仍保留在当前设备。数据库以外的后端、登录、真实 AI、图片识别和云同步仍不在范围内。

## 开源实现原则

官方核心只实现两条不依赖自有服务器的路径：完全离线使用，以及用户自带 Key 使用。账户、认证、权限、云存储和同步作为可选开放扩展，不成为应用启动或核心体验的依赖。

项目会维护版本化的开放扩展契约、OpenAPI/JSON Schema 和一致性测试，方便社区开发者或 AI Agent 实现兼容服务；官方核心不预置一套必须部署的业务后端。规范总则见 [`docs/05-开放扩展接口规范.md`](docs/05-开放扩展接口规范.md)。
